================================================================
  SPC 프로그램 데모 영상 녹화 가이드
  실행 프로그램: run_spc_desktop.bat (공정 안정성 점검)
================================================================

■ 빠른 시작 (자동 - 권장)
----------------------------------------------------------------
  1. prepare_spc_demo.bat        <- 샘플 데이터·폴더 준비
  2. record_spc_demo_auto.bat    <- run_spc_desktop.bat + 자동 녹화 (약 120초)
  3. data\output\recordings\     <- MP4 확인

■ 수동 녹화 (나레이션·연출 직접)
----------------------------------------------------------------
  1. prepare_spc_demo.bat
  2. record_spc_demo.bat
  3. Enter -> 녹화 시작
  4. 아래 시나리오대로 조작
  5. Enter -> 녹화 종료

■ 촬영 시나리오 (run_spc_desktop.bat - 8단계 마법사)
----------------------------------------------------------------
  [0:00] run_spc_desktop.bat 실행 -> "SPC 공정 안정성 점검" 창 표시
  [0:15] 1. 데이터 입력 - data\input\mes_data.xlsx 업로드
  [0:30] USL/LSL, 관리도 유형 등 분석 조건 확인
  [0:45] "공정 안정성 점검 시작" 클릭
  [1:00] 2. 데이터 분석 - 원본·채취 표본 확인
  [1:20] 3. 정규성 검정 - 히스토그램·QQ plot
  [1:40] 4. 관리도 해석 - 이상점·Western Electric 규칙
  [2:00] 5. 공정능력 평가 - Cp/Cpk, Case 1~4
  [2:30] 6. 결론 - 종합 리포트, Excel/PDF 다운로드

■ 녹화 팁
----------------------------------------------------------------
  - 해상도: 데스크톱 창을 최대화하면 선명도 향상
  - FPS: 기본 20 (scripts\record_spc_screen.py --fps 30)
  - 창 제목 "SPC 공정 안정성 점검" - 부분 일치로 자동 인식
  - Streamlit 서버 기동에 15~20초 소요 - 녹화 시작 전 대기

■ 기존 녹화 파일
----------------------------------------------------------------
  data\output\recordings\spc_demo_*.mp4

■ 문제 해결
----------------------------------------------------------------
  창을 찾을 수 없음 -> SPC 창을 앞으로 가져온 뒤 Enter
  패키지 오류       -> pip install mss opencv-python numpy
  앱 기동 실패      -> run_spc_desktop.bat 단독 실행으로 확인
  분석 실패         -> prepare_spc_demo.bat 재실행
